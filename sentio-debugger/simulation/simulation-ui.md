# Simulation UI

## From existed transaction

The simplest way to start a simulation is to click the simulator button as shown below, on a transaction that is been browsed.&#x20;

<figure><img src="../../.gitbook/assets/image.png" alt="" width="563"><figcaption></figcaption></figure>

In this case it will copy all the parameters from the existed transaction and you could do adjustment on top of it. Like  block number, block index, gas fee, block header, state etc.

<figure><img src="../../.gitbook/assets/image (1).png" alt="" width="375"><figcaption></figcaption></figure>

Click the simulate transaction button will save this run to simulation history of your project and show you the result, just like what you see from the normal debugger UI.

## Direct Build

You can also click simulator button on the left navigation bar and go to the simulator page which shows all the history simulations. Click simulation button on the right corner will pop the similar UI but without prepopulated transaction data.

<figure><img src="../../.gitbook/assets/image (2).png" alt=""><figcaption></figcaption></figure>
